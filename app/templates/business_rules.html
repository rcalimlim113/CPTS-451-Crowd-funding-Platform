# Business Rule SQL Enforcement Constraint Type
01 Unique emails for all users UNIQUE(email) UNIQUE
02 Role must be 'admin' or 'user' CHECK (role IN ('admin', 'user')) CHECK
03 Funding goal must be > 0 CHECK (funding_goal > 0) CHECK
04 End date must be after start date CHECK (end_date > start_date) CHECK
05 Donation amount must be > 0 CHECK (amount > 0) CHECK
06 Campaign must have a valid organizer FK organizer_id → users(user_id) FOREIGN KEY
07 Donation payment status must be recognized CHECK (payment_status IN ('pending','completed','failed','refunded')) CHECK
08 Campaign status must be a valid value CHECK (status IN ('pending','active','completed','cancelled')) CHECK
09 User core fields cannot be null NOT NULL on first_name, last_name, email, password_hash, role NOT NULL
